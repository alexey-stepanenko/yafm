# Скрипты

В этом каталоге располагаются скрипты для упрощения процесса разработки.

## develop.os

Основной скрипт для разработчика. Команды:

* **compile** - Загрузка конфигурации информационной базы из исходных кодов.
* **decompile** - Выгрузка конфигурации информационной базы в исходные коды.
* **help** - Показать справку.
* **version** - Показать версию приложения.

Команды `help` и `version` не имеют дополнительных параметров, а для команд `compile` и `decompile` список параметров одинаков.

### Параметры команд `compile` и `decompile`

* **-infobase** - Путь к информационной базе. Задается в формате параметра **IBConnectionString** запуска платформы. Например, для файловой базы: `File=""/home/user/repo/yafm/infobase"";`, для серверной: `Srvr=""srv-1c"";Ref=""infobase"";` (Необходимо удваивать символ двойных кавычек).
* **-infobase-user** - Пользователь информационной базы. Может быть пустым.
* **-infobase-user-password** - Пароль пользователя информационной базы. Может быть пустым.
* **-platform** - Использовать указанную версию платформы для загрузки, например `8.3.25`.
* **-repo** - Путь к каталогу репозитория. Абсолютный путь, например, для Linux `/home/user/repo/yafm`, (для Windows `D:\repo\yafm`)
* **-source** - Имя каталога с исходными кодами относительно каталога репозитория.  Сейчас это `src/cf` (для Windows необходимо использовать `src\cf`).
* **-test-extension-name** - Имя расширения с юнит-тестами. Сейчас это `ЮнитТесты`.
* **-tests** - Имя каталога с исходными кодами юнит-тестов относительно каталога репозитория. Сейчас это `tests/unit` (для Windows необходимо использовать `tests\unit`).
* **-conf** - Путь к файлу с настройками. Для упрощения запуска все вышеуказанные параметры можно оформить в виде JSON-файла, например:
    ``` json
    {
        "infobase": "File=\"/home/user/repo/yafm/infobase\";",
        "infobase-user": "Администратор",
        "infobase-user-password": "",
        "platform": "8.3.26",
        "repo": "/home/user/repo/yafm",
        "source": "src/cf",
        "tests": "tests/unit",
        "test-extension-name": "ЮнитТесты"
    }
    ```
    Имена параметров указываются **без** первого символа дефис.

    Если передать отдельно параметр в командной строке, его значение переопределит значение в файле настроек.

    При указании значений параметров содержащих символ двойной кавычки необходимо не удваивать, а экранировать с помощью символа обратной косой черты `\`.

### Загрузка

В данном режиме производится загрузка конфигурации информационной базы (параметр `-infobase`) из исходных кодов, размещенных в каталоге `-source`. Так же производится загрузка конфигурации расширения с юнит-тестами (параметр `-test-extension-name`) из каталога `-tests`. Обновление информационной базы не производится.

В корне репозитория располагается файл **compile.sh** для упрощения запуска режима загрузки.

### Выгрузка

В данном режиме производится выгрузка конфигурации информационной базы (параметр `-infobase`) в исходные коды, размещенные в каталоге `-source`. Так же производится выгрузка конфигурации расширения с юнит-тестами (параметр `-test-extension-name`) в каталог `-tests`. 

После выгрузки исходных кодов считывается текущая версия подсистемы **МодификацияФорм** (из файла `Configuration.xml`). Это значение записывается в версию расширения с тестами, а так же в функции `МФСлужебный.Версия()` и `МФИнструментыРазработчика.ВерсияИнструментовРазработчика()`.

Так же, если на момент запуска скрипта были изменены файлы, относящиеся к шаблону модуля с тестами (используется в обработке генерации автотестов), то производится сборка шаблона расширения с тестами и заменяется макет.

В корне репозитория располагается файл **decompile.sh** для упрощения запуска режима выгрузки.

## releases.os

Скрипт для работы с релизами. Команды:

* **create** - Создать релиз.
* **help** - Показать справку.
* **version** - Показать версию приложения.

Команды `help` и `version` не имеют дополнительных параметров.

## Создание релиза

Параметры команды:

* **-distrib-variants** - Список видов дистрибутивов для создания, разделенных символом `,`. Значение по-умолчанию `Поставка,РасширениеРазработчикаДляПоставки`.
* **-source** - Имя каталога с исходными кодами относительно каталога репозитория.  Сейчас это `src/cf` (для Windows необходимо использовать `src\cf`).
* **-releases** - Имя каталога с релизами относительно каталога репозитория.  Сейчас это `releases`.
* **-repo** - Путь к каталогу репозитория. Абсолютный путь, например, для Linux `/home/user/repo/yafm`, (для Windows `D:\repo\yafm`)
* **-create-number-release-dir** - Создавать отдельный каталог с номером релиза.
* **-conf** - Путь к файлу с настройками. Для упрощения запуска все вышеуказанные параметры можно оформить в виде JSON-файла, например:
    ``` json
    {
        "distrib-variants": "Поставка",
        "repo": "/home/user/repo/yafm",
        "source": "src/cf",
        "releases": "releases",
        "create-number-release-dir": true
    }
    ```
    Имена параметров указываются **без** первого символа дефис.

    Если передать отдельно параметр в командной строке, его значение переопределит значение в файле настроек.

    При указании значений параметров содержащих символ двойной кавычки необходимо не удваивать, а экранировать с помощью символа обратной косой черты `\`.

Скрипт предназначен для создания файла поставки (в параметре `distrib-variants` указано значение `Поставка`) и/или расширения разработчика (в параметре `distrib-variants` указано значение `РасширениеРазработчикаДляПоставки`). 

В корне репозитория располагается файл **create-release.sh** для упрощения запуска режима создания релиза.
